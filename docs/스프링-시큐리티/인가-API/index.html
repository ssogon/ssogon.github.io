인가 API

인가(authorization)란 사용자가 요청을 실행할 수 있는 권한이 있는지 확인하는 절차다.

권한 설정​

HttpSecurity의 authorizeRequests()를 호출하면 액세스 제한 기능이 활성화된다.

httpSecurity.antMatcher("/shop/**").authorizeRequests()
    .antMatchers("/shop/login", "/shop/users/**").permitAll()
    .antMatchers("/shop/mypage", "/shop/users/**").hasRole("USER")
    .antMatchers("/shop/admin/pay", "/shop/users/**").access("hasRole('ADMIN')")
    .antMatchers("/shop/admin/**", "/shop/users/**").access("hasRole('ADMIN') or hasRole('SYS')")
    .anyRequest().authenticated();

authorizeRequests()가 반환하는 ExpressionUrlAuthorizationConfigurer<HttpSecurity>를 통해 인가 표현식을 설정할 수 있다.

ExpressionUrlAuthorizationConfigurer​

ExpressionUrlAuthorizationConfigurer<HttpSecurity>는 인가 표현식의 대상을 지정한다.

메서드설명
antMatchers(String...)특정 URL들을 대상으로 하는 인가 표현식을 만든다.

각 메서드가 반환하는 ExpressionInterceptUrlRegistry로 표현식을 완성할 수 있다.

ExpressionInterceptUrlRegistry​

ExpressionInterceptUrlRegistry는 인가 표현식의 동작을 지정한다.

메서드설명
authenticated()인증된 사용자의 접근을 허용한다.
fullyAuthenticated()인증된 사용자의 접근을 허용한다. (remember-me 인증은 제외)
permitAll()모든 접근을 허용한다.
denyAll()모든 접근을 허용하지 않는다.
caution

설정 시 구체적인 경로가 먼저 오고 그 보다 큰 범위의 경로가 뒤에 오도록 해야 한다.
