자바독

자바독은 API의 개념적 공백을 명확히하고 접근하기 쉽게 만들어주는 역할을 한다.

/**
 * 아무 것도 하지 않습니다.
 */
 public void doNothing() {}

HTML 태그를 사용할 수 있다.

/**
 * 아무 것도 하지 않습니다.<br/>
 * 삭제될 예정입니다.
 */
 public void doNothing() {}

IDE의 자바독 렌더링 기능을 사용하면 태그가 적용된 모습을 볼 수 있을 것이다.

메타 태그​

@author​

@author 태그는 작성자를 명시한다.

/**
 * 아무 것도 하지 않습니다.
 *
 * @author scott
 */
 public void doNothing() {}

보통 소스 파일은 여러 사람의 손을 타기 때문에 @author보다는 커밋 히스토리를 신뢰하는게 더 낫다.

메서드 정의 태그​

@param​

@param 태그는 매개변수명과 매개변수에 대한 설명을 나타낸다.

/**
 * 주어진 메시지를 출력합니다.
 *
 * @param message 출력할 메시지
 */
public void print(String message) {
  System.out.println(message);
}

@return​

@return 태그는 반환 값에 대한 설명을 나타낸다.

/**
 * 빈 문자열을 얻습니다.
 *
 * @return 빈 문자열
 */
public String getBlank() {
  return "";
}

@throws​

@throws 태그는 발생할 수 있는 CheckedException의 클래스명과 설명을 나타낸다.

/**
 * 빈 파일을 만들거나 타임스탬프를 갱신 합니다.
 *
 * @param filePath 파일 경로
 * @throws IOException 입출력 예외가 발생한 경우
 */
public void touch(String filePath) throws IOException {
  Files.touch(new File(filePath));
}

@exception​

@throws 태그와 동일하며 유의미한 차이는 없다.

링크 태그​

@see​

@see 태그는 참조할 수 있는 레퍼런스를 나타낸다.
참조하는 대상을 import 했다면 패키지명은 생략할 수 있다.

/**
 * Service를 실행합니다.
 *
 * @see Service#execute
 */
public void execute() {
  service.execute();
}

인라인 태그​

{@link}​

{@link} 태그는 기본적으로 @see와 같으나 인라인하게 작성할 수 있다는 점이 다르다.

/**
 * {@link Service#execute}를 실행합니다.
 */
public void execute() {
  service.execute();
}

{@code}​

{@code} 태그는 예제 코드를 첨부하는 목적으로 사용된다.

/**
 * 매개변수로 주어진 문자열들을 하나로 연결합니다.
 *
 * {@code
 * String joinedString = Joiner.join("java", "doc");
 * }
 * @param parts 연결할 문자열들
 * @return 연결된 문자열
 */
public String join(String... parts) {
  return String.join("", parts);
}
