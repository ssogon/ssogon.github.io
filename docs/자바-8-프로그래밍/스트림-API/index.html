스트림 API

Stream API는 요소의 시퀀스를 functional하게 처리할 수 있는 방법을 제공한다.
스트림은 보통 Collection 인터페이스의 default 메서드인 stream()을 통해 생성된다.

List<String> names = Arrays.asList("scott", "michael", "holden");
Stream<String> nameStream = names.stream();

Stream.of(T...) 메서드를 사용하여 생성할 수도 있다.

Stream<String> nameStream = Stream.of("scott", "michael", "holden");

스트림 인스턴스​

생성된 스트림은 처리하는 원본 데이터를 변경하지 않으며 별도의 인스턴스에서 실행된다.

names.stream()
    .map(String::toUpperCase)
    .forEach(System.out::println); // SCOTT MICHAEL HOLDEN

names.forEach(System.out::println); // scott michael holden

재사용 불가​

스트림은 중개 연산자와 종료 연산자를 결합하여 파이프라인을 형성한다.
종료 연산이 수행된 후의 스트림 파이프라인은 재사용할 수 없다.

Stream<Widget> pipeline = widgets.stream().filter(b -> b.getColor() == RED);
int sum1 = pipeline.sum();
int sum2 = pipeline.mapToInt(b -> b.getWeight()).sum(); // stream has already been operated upon or closed

중개 연산자​

메서드 체이닝을 위해 Stream<T>을 반환하는 메서드를 중개 연산자라 말한다.
중개 연산자는 종료 연산자가 호출되기 전까지 실행되지 않는다는 특징이 있다.
이를 지연 실행(lazy evaluation)이라고도 한다.

Stream.of("scott", "michael", "holden")
    .map(System.out::println); // 실행되지 않기에 출력되는 값 또한 없음

filter​

스트림의 요소들 중 Predicate<T>를 만족하는 요소만 남긴다.

Stream.of("scott", "michael", "holden")
    .filter(name -> name.startWith("s"))
    .forEach(System.out::println); // scott

map​

스트림의 요소 T를 Function<T, R>을 이용해 R로 변경한다.

Stream.of("scott", "michael", "holden")
    .map(String::toUpperCase)
    .forEach(System.out::println); // SCOTT MICHAEL HOLDEN

flatMap​

스트림의 요소 T를 Function<T, Stream<R>>을 이용해 R로 변경한다.
여러 시퀀스를 하나의 단일 원소 스트림으로 처리하고자 할 때 유용하다.

Stream.of(Arrays.asList(1, 2, 3), Arrays.asList(4, 5, 6))
    .flatMap(Collection::stream)
    .forEach(System.out::println); // 1 2 3 4 5 6

sorted​

Comparable 인터페이스를 구현한 요소들을 compareTo(T) 메서드를 통해 정렬한다.
만약 요소들이 Comparable을 구현하지 않았다면 ClassCastException이 발생한다.

Stream.of("scott", "michael", "holden")
    .sorted()
    .forEach(System.out::println); // holden michael scott

Comparator<T>를 매개변수로 넘겨 정렬 방법을 직접 지정할 수도 있다.

Stream.of(memberA, memberB, memberC)
    .sorted(Comparator.comparing(member -> member.getName()))
    .forEach(System.out::println);

참고로, Comparator는 comparing(Function<T, R>) 이외에도 다양한 팩터리 메서드들을 제공한다.

distinct​

중복되는 스트림의 요소들을 equals(Object) 메서드를 이용해 제거한다.

IntStream.of(1, 2, 2, 3, 4, 4, 5)
    .distinct()
    .forEach(System.out::println); // 1 2 3 4 5

peek​

스트림의 요소들을 Consumer<T>를 통해 순회한다.

IntStream.rangeClosed(1, 5)
    .peek(System.out::println) // 1 2 3 4 5
    .forEach(i -> {});

limit​

스트림의 요소들을 특정 개수만큼 제한한다.

IntStream.rangeClosed(1, 100)
    .limit(5)
    .forEach(System.out::println); // 1 2 3 4 5

skip​

스트림의 요소들을 특정 개수만큼 건너뛴다.

IntStream.rangeClosed(1, 100)
    .skip(95)
    .forEach(System.out::println); // 96 97 98 99 100

종료 연산자​

더 이상 Stream<T>을 반환하지 않고 최종 결과를 반환하는 메서드를 종료 연산자라 말한다.

forEach​

스트림의 요소들을 Consumer<T>를 통해 순회한다.

Stream.of("scott", "michael", "holden")
    .forEach(System.out::println); // scott michael holden

anyMatch​

스트림의 요소들 중 Predicate<T>를 만족하는 요소가 하나라도 있다면 true를 반환한다.

Stream.of("scott", "michael", "holden")
    .anyMatch(name -> name.startsWith("s")); // true

allMatch​

스트림의 모든 요소가 Predicate<T>를 만족한다면 true를 반환한다.

Stream.of("scott", "michael", "holden")
    .allMatch(name -> name.startsWith("s")); // false

noneMatch​

스트림의 모든 요소가 Predicate<T>를 만족하지 못한다면 true를 반환한다.

Stream.of("scott", "michael", "holden")
    .noneMatch(name -> name.startsWith("s")); // false

findFirst​

스트림의 첫 번째 요소를 Optional<T> 형태로 반환한다.

Stream.of("scott", "michael", "holden")
    .findFirst()
    .get(); // scott

findAny​

스트림의 아무 요소를 Optional<T> 형태로 반환한다.
병렬으로 처리하지 않는 이상 첫 번째 요소가 반환될 가능성이 높지만 이를 보장하지는 않는다.

Stream.of("scott", "michael", "holden")
    .findAny()
    .get(); // 예상할 수 없음

reduce​

스트림의 요소들을 BinaryOperator<T>를 이용해 하나로 뭉쳐 Optional<T> 형태로 반환한다.

IntStream.rangeClosed(1, 10)
    .reduce(Integer::sum)
    .getAsInt(); // 55

count​

스트림에 포함된 요소들의 개수를 반환한다.

Stream.of("scott", "michael", "holden")
    .count(); // 3

max​

스트림의 요소들 중 가장 큰 요소를 Optional<T> 형태로 반환한다.
비교에 사용할 Comparator<T>를 필요로 한다.

Stream.of("scott", "michael", "holden")
    .max(Comparator.comparing(name -> name))
    .get(); // scott

IntStream처럼 primitive 타입을 위한 스트림은 별도의 Comparator<T>를 필요로 하지 않는다.
primitive 타입이기 때문에 Optional<T> 형태로 반환하지도 않는다.

IntStream.rangeClosed(1, 5)
    .max(); // 5

min​

스트림의 요소들 중 가장 작은 요소를 Optional<T> 형태로 반환한다.
비교에 사용할 Comparator<T>를 필요로 한다.

Stream.of("scott", "michael", "holden")
    .min(Comparator.comparing(name -> name))
    .get(); // holden

IntStream과 같은 primitive 타입을 위한 스트림은 Comparator<T>를 필요로 하지도 않고 Optional<T> 형태로 반환하지도 않는다.

IntStream.rangeClosed(1, 5)
    .min(); // 1

toArray​

스트림을 배열 형태로 변환하여 반환한다.
배열을 생성하는데 사용할 IntFunction<R>을 요구한다.
IntFunction<R>의 인자로 들어올 int는 스트림 요소의 개수이고 반환 값 R은 배열이 되어야 한다.

String[] names = Stream.of("scott", "michael", "holden")
    .toArray(String[]::new);

collect​

스트림의 요소들을 Collector<T, A, R>를 이용하여 수집한다.
컬렉터를 직접 작성하는 경우는 드물고 보통 Collectors에서 제공하는 컬렉터를 사용한다.

toList​

스트림을 List<T> 형태로 변환하여 반환한다.

List<String> names = Stream.of("scott", "michael", "holden")
    .collect(Collectors.toList());

toSet​

스트림을 Set<E> 형태로 변환하여 반환합니다.

Set<String> names = Stream.of("scott", "michael", "holden")
    .collect(Collectors.toSet());

toMap​

스트림을 Map<K, V> 형태로 변환하여 반환한다.
요소 중 어떤 것을 키로 할 것이고 어떤 것을 값으로 취급할 것인지를 알려주기 위해 Function<T, K>, Function<T, V>을 매개변수로 넘겨야 한다.

Map<Integer, String> idToName = ImmutableMap.of(1, "scott", 2, "michael", 3, "holden")
    .entrySet()
    .stream()
    .collect(Collectors.toMap(Map.Entry::getKey, Map.Entry::getValue));

키가 충돌할 경우 어떤 값을 사용할 것인지 결정하는 BinaryOperator<T>를 지정할 수도 있다.

Map<Integer, String> idToName = ImmutableMap.of(1, "scott", 1, "michael", 1, "holden")
    .entrySet()
    .stream()
    .collect(Collectors.toMap(Map.Entry::getKey, Map.Entry::getValue, (left, right) -> left));

toCollection​

스트림을 특정 Collection 형태로 변환하여 반환한다.
배열을 생성하는데 사용할 Supplier<R>을 필요로 한다.

List<String> names = Stream.of("scott", "michael", "holden")
    .collect(Collectors.toCollection(LinkedList::new));

joining​

문자열 스트림을 하나의 문자열로 결합하여 반환한다.
이 컬렉터는 T가 CharSequence 타입이어야 사용할 수 있다.

Stream.of("scott", "michael", "holden")
    .collect(Collectors.joining("\n")); // scott michael holden

groupingBy​

스트림의 T를 그룹핑하여 Map<T, List<T>> 형태로 반환한다.
인자로 키 분류기, 맵 생성자, 최종 결과 Collector를 필요로 한다.

Map<String, List<String>> groupByName = Stream.of("scott", "michael", "holden", "scott")
    .collect(Collectors.groupingBy(name -> name, HashMap::new, Collectors.toList()));
// {michael=[michael], scott=[scott, scott], holden=[holden]}

partitioningBy​

groupingBy와 유사하지만 Predicate<T> 결과를 기준으로 그룹핑한다는 점이 다르다.
한 쪽 결과만 필요하다면 filter를 사용하는 것이 더 나을 것이다.

Map<Boolean, List<String>> partitionByName = Stream.of("scott", "michael", "holden", "scott")
    .collect(Collectors.partitioningBy(name -> name.contains("e")));
// {true=[michael, holden], false=[scott, scott]}

primitive 타입 스트림​

primitive 타입을 스트림의 요소로 사용하게 되면 오토 박싱이 일어날 수 밖에 없다.

Stream<Integer> stream = Stream.of(1, 2, 3); // auto boxing

이러한 박싱 오버헤드를 없애고자 primitive 타입을 위한 스트림을 따로 제공하고 있다.

primitive 타입 스트림primitive 타입
IntStreamint
LongStreamlong
DoubleStreamdouble

of​

가변 인자를 요소로 갖는 primitive 타입 스트림을 생성한다.

IntStream stream = IntStream.of(1, 2, 3);

range​

특정 범위로 primitive 타입 스트림을 생성한다.
마지막 숫자를 포함하지 않는 범위다.

IntStream stream = IntStream.range(1, 5); // [1, 2, 3, 4]

rangeClosed​

특정 범위로 primitive 타입 스트림을 생성한다.
마지막 숫자를 포함하는 범위다.

IntStream stream = IntStream.rangeClosed(1, 5); // [1, 2, 3, 4, 5]

병렬 스트림​

병렬 처리를 수행하는 스트림은 Collection 인터페이스의 default 메서드인 parallelStream()을 통해 생성된다.

Arrays.asList("scott", "michael", "holden")
    .parallelStream()
    .forEach(System.out::println); // 예상할 수 없음

스레드 생성, 병렬 처리 후 수집 비용, 컨텍스트 스위칭 등 병렬 처리라고 마냥 좋은 것은 아니다.
방대한 양의 데이터를 처리해야할 경우에만 병렬 처리를 고려해볼 수 있겠다.

ForkJoinPool​

병렬 스트림은 Fork/Join 프레임워크를 사용하며, 하나의 태스크를 여러 스레드로 분할(fork)하고 결과를 합치는(join) 과정을 통해 최종 결과물을 만들어 낸다.

문제는 이 Fork/Join 프레임워크가 하나의 전역 스레드풀만을 사용한다는 것이다.

ForkJoinPool.commonPool(); // global

기본적으로 설정되는 스레드 풀의 크기는 CPU 코어 수와 같으며, 만약 현재 PC의 CPU 코어 수가 4개라면 총 4개까지의 스레드만 동시에 처리할 수 있다.
만약 추가적인 병렬 스트림 요청이 들어온다면 다른 스레드가 종료될 때까지 대기해야 하는 상황에 놓여지게 될 것이다.
스레드 풀의 크기는 시스템 프로퍼티로 변경할 수 있지만 근본적인 해결책은 되지 못한다.

System.setProperty("java.util.concurrent.ForkJoinPool.common.parallelism", "20");

로컬 ForkJoinPool​

위에서 살펴본 전역 ForkJoinPool의 문제점은 로컬 ForkJoinPool을 생성함으로써 해결할 수 있다.

ForkJoinPool forkJoinPool = new ForkJoinPool(4);
forkJoinPool.submit(() ->
    Arrays.asList("scott", "michael", "holden")
        .parallelStream()
        .forEach(System.out::println)
).get();
