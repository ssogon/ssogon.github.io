들어가며

본 문서는 백기선 님의 더 자바, Java 8을 수강하고 정리한 것임을 밝힌다.
개인적으로 찾아본 내용들도 덧붙였기 때문에 강의 내용과는 조금 상이할 수 있다.

Inflearn: 공부한 것들 정리해서 블로깅해도 되나요?

변화점​

자바 8은 자바 역사상 가장 큰 변환점이라 해도 과언이 아니다.
구문 변경, 새로운 타입 및 메서드 등 많은 기능들이 추가되었지만 이 기능들을 직접 이용하지 않더라도 애플리케이션에 도움이 되는 변경 사항들도 포함되어 있다.

빠른 퍼포먼스​

자바 8을 이용한 애플리케이션은 특별한 작업이나 튜닝 없이도 어느 정도 속도가 향상될 수 있다.
특정 JVM의 구현에 맞게 조정된 애플리케이션에는 적용되지 않을 수 있지만, 자바 8이 더 나은 성능을 발휘할 수 있는 데는 아래와 같은 이유들이 있다.

  • HashMap<K, V>이 키 충돌로 인해 버킷이 너무 커질 경우 트리 맵을 사용하도록 하도록 변경되었다.
  • 기존 PermGen 영역을 제거하고 Metaspace 영역을 도입함으로써 가비지 컬렉터의 성능을 향상시켰다.
  • 포크/조인 프레임워크의 성능을 개선하기 위한 작업이 진행되었다.

애플리케이션 속도 개선을 위해 기존 코드를 변경할 필요 없이 자바 버전만 업그레이드하면 된다는 사실은 매력적으로 다가온다.

보일러 플레이트 코드 감소​

자바는 항상 보일러 플레이트 코드가 무겁다는 지적을 받아왔다.
자바 8은 새로운 API를 설계할 때 보다 효율적인 스타일을 수용하여 이러한 문제들을 일부 해소하였다.

// jdk 1.8 이전
Runnable runnable = new Runnable() {
  @Override
  public void run() {
    System.out.println("hello, world");
  }
};
// jdk 1.8 이후
Runnable runnable = () -> System.out.println("hello, world");
