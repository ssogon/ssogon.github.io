InternetDomainName​

InternetDomainName은 도메인 네임을 분석하고 조작하고자 할 때 유용하게 쓰일 수 있다.

Guava: InternetDomainName Introduction

공개 접미사와 최상위 개인 도메인​

InternetDomainName은 RFC 명세서에 따라 구문상 유효함을 보장한다.
하지만 이것이 인터넷의 실제 도메인과 일치한다고 보장하지는 않는다.
도메인에 대한 실제 lookup을 수행하지 않고는 불가능한 일이기 때문이다.

그럼에도 주어진 도메인 이름이 인터넷의 실제 도메인을 나타낼 수 있는지 판단하는 것은 매우 유용하다.
이를 위해 InternetDomainName는 Mozilla Foundation에서 제공하는 PSL(Public Suffix List)을 이용한다.
기본적인 용례로 domain.hasPublicSuffix()가 true면 해당 도메인은 인터넷의 실제 도메인에 해당할 수 있다.

참고로 아래 세 가지 용어들은 모두 독립적인 개념이다.

  • 등록 접미사: com, co.uk처럼 도메인 이름 등록 기관을 통해 제어되는 도메인이다.
  • 공개 접미사: 등록 접미사의 상위 집합으로, blogspot.com처럼 PSL에는 포함되어 있지만 도메인 이름 등록 기관에 의해서는 제어되지는 않는 접미사도 있다.
  • TLD(Top-Level Domain): com, au처럼 자식이 없는 단일 라벨 도메인이다.
예시등록 접미사공개 접미사TLD(Top-Level Domain)
comOOO
auXXO
com.auOOX
blogspot.comXOX
squerfXXX

등록 접미사와 TLD는 명확한 정의를 가지고 있는 반면 공개 접미사는 비교적 모호하다.
공개 접미사는 PSL을 신뢰하는 수 밖에 없다.

https://publicsuffix.org/list/public_suffix_list.dat

다시 InternetDomainName 이야기로 돌아와서, 특정 도메인이 신뢰할 수 있는 인터넷 도메인인지 확인하려면 hasPublicSuffix() 메서드를 사용하기만 하면 된다.
하지만 문제는 최상위 개인 도메인(Top Private Domain)을 식별하거나 추출하는 과정에서 발생한다.

기술적인 관점에서 볼 때 최상위 개인 도메인은 단순히 하나의 하위 도메인 뒤에 공개 접미사가 붙는 형태로 구성된 도메인이다.
예를 들어 www.foo.co.uk의 공개 접미사는 co.uk고, 최상위 개인 도메인은 foo.co.uk이다.

InternetDomainName internetDomainName = InternetDomainName.from("www.foo.co.uk");
internetDomainName.publicSuffix(); // co.uk
internetDomainName.registrySuffix(); // co.uk
internetDomainName.topPrivateDomain(); // foo.co.uk

하위 도메인에서 진짜 도메인 또는 소유자 도메인(Owner Domain)을 찾고 싶을 수도 있다.
mail.google.com의 소유자 도메인은 google.com이다.

소유자 도메인이 최상위 개인 도메인과 같은 것처럼 보일 수 있으나 실상은 그렇지 않다.
blogspot.com은 그 자체로 공개 접미사이면서 소유자 도메인이기도 하다.
따라서 최상위 개인 도메인을 조회하는 topPrivateDomain()로는 소유자 도메인을 추출할 수 없다.

InternetDomainName internetDomainName = InternetDomainName.from("foo.blogspot.com");
internetDomainName.topPrivateDomain(); // foo.blogspot.com

정답은 하나의 하위 도메인 뒤에 등록 접미사가 붙는 형태로 구성된 도메인을 확인하는 것이다.

InternetDomainName internetDomainName = InternetDomainName.from("foo.blogspot.com");
internetDomainName.topDomainUnderRegistrySuffix(); // blogspot.com

다음은 각 접미사들 간의 차이점을 요약한 것이다.

google.comcomcomgoogle.comgoogle.com
co.ukco.ukco.ukN/AN/A
www.google.co.ukco.ukco.ukgoogle.co.ukgoogle.co.uk
foo.blogspot.comcomblogspot.comfoo.blogspot.comblogspot.com
google.invalidN/AN/AN/AN/A
  • 등록 접미사, 공개 접미사, 최상위 개인 도메인, 소유자 도메인, TLD는 서로 같지 않다.
  • 공개 접미사는 도메인 유효성 검사 및 슈퍼쿠키 억제를 위한 목적으로 사용된다.
  • 공개 접미사는 사람에 의해 정의되며 예측할 수 없이 변경될 수 있다.
  • 공개 접미사와 소유권 사이의 매핑은 따로 정의되어 있지 않는다.
  • 등록 접미사는 도메인의 소유권을 결정하는 데 도움이 될 수 있다.
    하지만 이 정보는 대부분 공개 접미사보다 덜 적합하다.
  • InternetDomainName으로 특정 도메인이 인터넷의 실제 도메인이 될 수 있는지 판단할 수 있다.
    그리고 도메인의 어느 부분에 쿠키를 설정할 수 있는지도 확인할 수 있다.
  • InternetDomainName은 도메인이 될 수 있는지만 판단 가능할 뿐, 실제로 존재하는지는 확인할 수 없다.

참고로 공개 접미사 목록은 PSL을 실시간으로 읽어오는 것이 아니라 PublicSuffixPatterns에 미리 정의해둔 목록을 참조한다.
이는 PSL이 갱신됨에 따라 실패 사례가 변경될 수도 있음을 의미한다.
