그래프

노드와 노드 사이의 관계를 표현하는 guava의 그래프에 대해 설명한다.

Guava: Graphs, Explained

정의​

그래프는 노드와 양 노드를 이어주는 간선으로 구성되어 있다.
만약 간선의 시작점과 끝점을 지정한다면 그 간선은 방향성을 가지게 된다.

graph.addEdge(nodeU, nodeV, edgeUV); // nodeU와 nodeV를 잇는 edgeUV를 그래프에 추가한다.
  • 방향 그래프는 비대칭 관계를 표현할 수 있다. (nodeU → nodeV)
  • 무방향 그래프는 대칭 관계를 표현할 수 있다. (nodeU ↔ nodeV)

방향 그래프에서 시작점과 끝점이 같은 간선을 평행하다고 말한다.
무방향 그래프는 방향성을 가지지 않으므로 양 노드만 같으면 평행하다고 말한다.

directedGraph.addEdge(nodeU, nodeV, edgeUV_a); // nodeU -> nodeV
directedGraph.addEdge(nodeU, nodeV, edgeUV_b); // nodeU -> nodeV
directedGraph.addEdge(nodeV, nodeU, edgeVU); // nodeV -> nodeU

undirectedGraph.addEdge(nodeU, nodeV, edgeUV_a); // nodeU <-> nodeV
undirectedGraph.addEdge(nodeU, nodeV, edgeUV_b); // nodeU <-> nodeV
undirectedGraph.addEdge(nodeV, nodeU, edgeVU); // nodeV <-> nodeU
  • directedGraph의 edgeUV_a와 edgeUV_b는 서로 평행하며, edgeVU와는 평행하지 않는다.
  • undirectedGraph의 edgeUV_a, edgeUV_b, edgeVU는 모두 평행하다.

Guava: Definitions

지원 범위​

com.google.common.graph 패키지가 지원하는 그래프는 다음과 같다.

  • 방향 그래프
  • 무방향 그래프
  • 값을 가지는 노드 또는 간선
  • 셀프 루프를 허용하거나 허용하지 않는 그래프
  • 평행 간선을 허용하거나 허용하지 않는 그래프
  • 노드 또는 간선이 삽입된 순서를 가지거나 가지지 않는 그래프

방향 간선과 무방향 간선이 양립하는 그래프는 지원하지 않는다.

Guava: Capabilities

그래프 유형​

com.google.common.graph 패키지에는 간선의 표현 방식에 따라 구분되는 Graph<N>, ValueGraph<N, V>, Network<N, E> 세 개의 최상위 그래프 인터페이스가 있다.
각 최상위 인터페이스는 공통적으로 SuccessorsFunction와 PredecessorsFunction 인터페이스를 상속한다.

Grpah​

Graph<N>는 가장 간단하고 기본적인 그래프 유형이다.
노드 간 관계를 처리하기 위한 저레벨 메서드들을 정의하고 있다.

Guava: Graph

빌더​

GraphBuilder의 정적 메서드를 통해서 Graph<N>를 생성할 수 있다.

MutableGraph<Integer> graph = GraphBuilder.undirected().build();

인스턴스 메서드​

인스턴스 메서드설명
Set<N> successor(N)특정 노드가 가리키는 노드들을 반환한다.
Set<N> predecessors(N)특정 노드를 가리키는 노드들을 반환한다.
Set<N> adjacentNodes(N)특정 노드와 인접한 노드들을 반환한다.
int outDegree(N)특정 노드가 가리키는 노드들의 수를 반환한다.
int inDegree(N)특정 노드를 가리키는 노드들의 수를 반환한다.
int degree(N)특정 노드와 인접한 노드들의 수를 반환한다.
Set<EndpointPair<N>> incidentEdges(N)특정 노드와 연결된 간선들을 반환한다.

무방향 그래프일 경우 successor(N)와 predecessors(N) 메서드는 adjacentNodes(N) 메서드와 동일하게 동작하고, outDegree(N)와 inDegree(N) 메서드는 degree(N) 메서드와 동일하게 동작한다.

ValueGraph​

ValueGraph<N, V>는 각 간선에 값 V를 가지고 있는 그래프다.
Graph<N>의 노드 관련 메서드들을 그대로 포함하며, 값 V를 제어하는 메서드들을 가지고 있다.

Guava: ValueGraph

빌더​

ValueGraphBuilder의 정적 메서드를 통해서 ValueGraph<N, V>를 생성할 수 있다.

MutableValueGraph<City, Distance> roads = ValueGraphBuilder.directed()
    .incidentEdgeOrder(ElementOrder.stable())
    .build();

인스턴스 메서드​

인스턴스 메서드설명

값이 없을 경우 디폴트로 지정한 값을 반환한다.

Network​

Network<N, E>는 간선 E를 노드와 같은 1급 고유 객체로 취급하는 그래프다.
Graph<N>의 노드 관련 메서드들을 포괄하면서, 노드와 간선간의 관계를 다루는 메서드들을 가지고 있다.

Guava: Network

빌더​

NetworkBuilder의 정적 메서드를 통해서 Network<N, E>를 생성할 수 있다.

MutableNetwork<Webpage, Link> webSnapshot = NetworkBuilder.directed()
    .allowsParallelEdges(true)
    .nodeOrder(ElementOrder.natural())
    .expectedNodeCount(100000)
    .expectedEdgeCount(1000000)
    .build();

인스턴스 메서드​

인스턴스 메서드설명
Set<E> outEdges(N)특정 노드에서 시작하는 간선들을 반환한다.
Set<E> inEdges​(N)특정 노드에서 끝나는 간선들을 반환한다.
EndpointPair<N> incidentNodes(E)특정 간선의 양 노드들을 반환한다.
Set<E> edgesConnecting​(N nodeU, N nodeV)특정 두 노드 사이를 잇는 간선들을 반환한다.
Graph<N> asGraph()현재 Network<N, E>를 Graph<N>로 변환하여 반환한다.

무방향 그래프일 경우 outEdges(N)와 inEdges​(N) 메서드는 incidentEdges​(N) 메서드와 동일하게 동작한다.

올바른 그래프 유형을 선택하는 방법​

세 그래프 유형 간의 근본적인 차이점은 간선을 표현하는 방법에 있다.

  • Graph<N>는 노드 간 익명 간선으로 연결되어 있다.
    각 노드 간 최대 하나의 간선으로 연결되어 있고 간선에 부가적인 정보가 필요하지 않은 경우 사용된다.
  • ValueGraph<N, V>는 간선에 고유하거나 고유하지 않을 수 있는 값을 가지고 있다.
    각 노드 간 최대 하나의 간선으로 연결되어 있고 간선에 부가적인 정보가 필요한 경우 사용된다.
  • Network<N, E>의 간선은 노드와 같은 1급 고유 객체다.
    각 간선이 고유해야 하고 이를 참조하는 쿼리를 실행할 수 있어야 할 때 사용된다.

Guava: Choosing the right graph typee

인스턴스 빌드​

com.google.common.graph 패키지가 제공하는 그래프의 구현 클래스는 설계상 공개되지 않고 빌드 클래스를 이용해 생성된다.
이렇게 하면 사용자가 알아야 하는 공개 유형의 수가 줄어들고, 그래프를 만드려는 사용자에게 부담을 주지 않으면서, 빌드된 구현이 제공하는 다양한 기능들을 보다 쉽게 탐색할 수 있다.

// Creating mutable graphs
MutableGraph<Integer> graph = GraphBuilder.undirected().build();

그래프 빌더의 인스턴스를 얻는 방법에는 두 가지가 있다.

  • 정적 메서드 directed() 또는 undirected() 메서드를 호출하여 방향 그래프 또는 무방향 그래프를 생성하는 빌더 인스턴스를 반환받는다.
  • 정적 메서드 from() 메서드를 호출해 기존 그래프 인스턴스를 기반으로 새로운 그래프를 생성하는 빌더 인스턴스를 반환받는다.

빌더 인스턴스에서 build() 메서드를 호출해 빌드하기 전 immutable() 메서드를 호출하면 불변 그래프를 만들 수도 있다.

// Creating an immutable graph
ImmutableGraph<Country> countryAdjacencyGraph =
    GraphBuilder.undirected()
        .<Country>immutable()
        .putEdge(FRANCE, GERMANY)
        .putEdge(FRANCE, BELGIUM)
        .putEdge(GERMANY, BELGIUM)
        .addNode(ICELAND)
        .build();

제약 조건과 최적화 힌트​

제약 조건이란 빌더에서 생성된 그래프가 충족해야 하는 동작 및 속성을 말한다.

  • directed(), undirected(): 방향 여부를 지정한다.
  • allowsSelfLoops(boolean): 셀프 루프 여부를 지정한다.
  • nodeOrder(ElementOrder<N>): 노드의 정렬 방식을 지정한다.
  • incidentEdgeOrder(ElementOrder<N>): 간선의 정렬 방식을 지정한다.

최적화 힌트는 효율성을 높이기 위해 선택적으로 사용될 수 있다.

  • expectedNodeCount​(int): 예상되는 노드의 수를 지정한다.
  • expectedEdgeCount​(int): 예상되는 간선의 수를 지정한다.

불변 여부​

가변 그래프​

각 그래프 유형은 그래프를 수정할 수 있는 MutableGraph<N>, MutableValueGraph<N, V>, MutableNetwork<N, E> 하위 유형과 대응된다.

  • 노드 추가 및 제거
    • addNode(node)
    • removeNode(node)
  • 간선 추가 및 제거
    • MutableGraph<N>
      • putEdge(nodeU, nodeV)
      • removeEdge(nodeU, nodeV)
    • MutableValueGraph<N, V>
      • putEdgeValue(nodeU, nodeV, value)
      • removeEdge(nodeU, nodeV)
    • MutableNetwork<N, E>
      • addEdge(nodeU, nodeV, edge)
      • removeEdge(edge)

Guava: Mutable* types

불변 그래프​

각 그래프 유형은 그래프를 수정할 수 없는 ImmutableGraph<N>, ImmutableValueGraph<N, V>, ImmutableNetwork<N, E> 하위 유형과 대응된다.
불변 그래프는 guava의 다른 불변 컬렉션들과는 달리 Collection을 구현한 것이 아니기 때문에 수정 메서드를 처음부터 정의하지도 않았다. (UnsupportedOperationException을 마주할 일이 없다는 뜻이다.)

불변 그래프의 인스턴스는 GraphBuilder의 immutable() 메서드를 통해 생성할 수 있다.

ImmutableGraph<Country> immutableGraph1 =
    GraphBuilder.undirected()
        .<Country>immutable()
        .putEdge(FRANCE, GERMANY)
        .putEdge(FRANCE, BELGIUM)
        .putEdge(GERMANY, BELGIUM)
        .addNode(ICELAND)
        .build();

또는 불변 그래프 구현체의 copyOf() 메서드를 통해서도 생성할 수 있다.

ImmutableGraph<Integer> immutableGraph2 = ImmutableGraph.copyOf(otherGraph);

보증 사항​

불변 그래프는 다음 사항들을 보증한다.

  • 얕은 불변성: 요소를 추가하거나 제거하거나 변경할 수 없다.
  • 결정론적 반복: 요소의 반복 순서는 입력된 순서와 동일하다.
  • 스레드 안전: 여러 스레드에서 동시에 해당 그래프에 접근해도 안전하다.
  • 무결성: 해당 클래스는 패키지 외부에서 서브클래싱될 수 없다.

예시 코드​

다음은 무방향 그래프를 노드 단위로 탐색하는 예이다.

// Return all nodes reachable by traversing 2 edges starting from "node"
// (ignoring edge direction and edge weights, if any, and not including "node").
Set<N> getTwoHopNeighbors(Graph<N> graph, N node) {
  Set<N> twoHopNeighbors = new HashSet<>();
  for (N neighbor : graph.adjacentNodes(node)) {
    twoHopNeighbors.addAll(graph.adjacentNodes(neighbor));
  }
  twoHopNeighbors.remove(node);
  return twoHopNeighbors;
}

이어서 방향 그래프를 간선 단위로 탐색하는 예이다.

// Update the shortest-path weighted distances of the successors to "node"
// in a directed Network (inner loop of Dijkstra's algorithm)
// given a known distance for {@code node} stored in a {@code Map<N, Double>},
// and a {@code Function<E, Double>} for retrieving a weight for an edge.
void updateDistancesFrom(Network<N, E> network, N node) {
  double nodeDistance = distances.get(node);
  for (E outEdge : network.outEdges(node)) {
    N target = network.target(outEdge);
    double targetDistance = nodeDistance + edgeWeights.apply(outEdge);
    if (targetDistance < distances.getOrDefault(target, Double.MAX_VALUE)) {
      distances.put(target, targetDistance);
    }
  }
}
