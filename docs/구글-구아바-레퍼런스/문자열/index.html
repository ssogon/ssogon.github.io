문자열

기존 자바의 문자열 유틸리티들이 어떤 문제점을 가지고 있었는지 살펴보고, 이를 개선시킨 guava의 문자열 유틸리티들을 알아보자.

Joiner​

기존 java.util.StringJoiner는 다음과 같은 문제점을 가지고 있다.

  • 가변 인자를 지원하지 않아 add(CharSequence) 메서드를 여러 번 호출해야 한다.
  • add(CharSequence) 메서드의 인자로 null을 넘기면 대신 문자열 "null"을 넣는다.

반면 Joiner는 보다 매끄럽게 문자열들을 결합할 수 있는 방법을 제공한다.

Joiner joiner = Joiner.on("; ").skipNulls();
joiner.join("Harry", null, "Ron", "Hermione"); // Harry; Ron; Hermione

Guava: Joiner

정적 메서드​

정적 메서드설명
Joiner on(char)특정 문자를 구분자로 가지는 Joiner를 반환한다.
Joiner on(String)특정 문자열을 구분자로 가지는 Joiner를 반환한다.

String join(Object...)매개변수들을 separator로 합친 문자열을 반환한다.

Guava Docs: Joiner

불변성​

Joiner 인스턴스는 불변하기 때문에 thread-safe하며 static final 상수로 사용할 수 있다.
skipNulls(), useForNull(String) 메서드는 this를 반환하는 것이 아니라 항상 새로운 Joiner를 만들어 반환하는 것이다.

Guava: Joiner

Splitter​

기존 java.lang.String.split() 메서드는 다음과 같은 문제점을 가지고 있다.

  • 꼬리 칸의 빈 문자열을 무시하기 때문에 예측하기 어렵다.
  • String[]을 반환하기 때문에 다루기가 힘들다.

또한 StringTokenizer는 nextToken() 메서드로 하나씩 가져와야 하는 불편함이 있다.

StringTokenizer tokenizer = new StringTokenizer("foo,bar,qux", ",");
while(tokenizer.hasMoreTokens()) {
  String token = tokenizer.nextToken();
}

반면 guava의 Splitter는 이러한 문제들을 유연하게 풀어낸다.

Splitter.on(",")
    .trimResults()
    .omitEmptyStrings()
    .split("foo,bar,,   qux"); // ["foo", "bar", "qux"]

Guava: Splitter

Splitter on(String)특정 문자열을 구분자로 가지는 Splitter를 반환한다.
Splitter on(Pattern)특정 정규식을 구분자로 가지는 Splitter를 반환한다.

마지막 조각은 지정한 길이보다 작을 수 있지만 비어있지는 않다.

CharMatcher​

CharMatcher는 문자열이 특정 조건에 매칭되는지 검증하거나 매칭된 문자열을 치환하는 방법을 제공한다.

Guava: CharMatcher

정적 메서드설명

CharMatcher none()아무런 문자도 매칭시키지 않는 CharMatcher를 반환한다.
CharMatcher whitespace()공백 문자를 매칭시키는 CharMatcher를 반환한다.

CharMatcher javaIsoControl()ISO 제어문자를 매칭시키는 CharMatcher를 반환한다.
CharMatcher ascii()ASCII 문자를 매칭시키는 CharMatcher를 반환한다.
CharMatcher anyOf(CharSequence)특정 문자열을 매칭시키는 CharMatcher를 반환한다.
CharMatcher is(char)특정 문자를 매칭시키는 CharMatcher를 반환한다.

Escaper​

Escaper는 문자열에 포함된 특정 문자를 치환하는 클래스로, Escapers의 정적 메서드에 의해 생성된다.

Escaper htmlEscaper = Escapers.builder()
    .addEscape('"', "&quot;")
    .addEscape('\'', "&#39;")
    .addEscape('&', "&amp;")
    .addEscape('<', "&lt;")
    .addEscape('>', "&gt;")
    .build();
htmlEscaper.escape("<p>text</p>"); // &lt;p&gt;text&lt;/p&gt;

HTML 특수 문자를 치환하는 Escaper는 HtmlEscapers에 미리 준비되어 있다.

Escaper htmlEscaper = HtmlEscapers.htmlEscaper();
htmlEscaper.escape("<p>text</p>"); // &lt;p&gt;text&lt;/p&gt;

Strings​

Strings는 범용적으로 사용할 수 있는 정적 메서드들을 모아놓은 유틸리티 클래스다.

Guava: Strings

정적 메서드​

정적 메서드설명
String commonPrefix(CharSequence a, CharSequence b)a, b의 공통된 접두어를 반환한다.
String commonSuffix(CharSequence a, CharSequence b)a, b의 공통된 접미어를 반환한다.
String emptyToNull(String str)str이 빈 문자열이라면 null을 반환하고, 그 외의 경우 str을 그대로 반환한다.
boolean isNullOrEmpty(String str)str이 null이거나 빈 문자열이라면 true를 반환한다.
String lenientFormat(String template, Object... args)template에 포함된 "%s"를 가변인자 args로 각각 치환한다.
String의 format 메서드와는 달리 "%s"의 개수와 args의 길이가 일치하지 않더라도 MissingFormatArgumentException이 발생하지 않는다.
String nullToEmpty(String str)str이 null이라면 빈 문자열을 반환하고, 그 외의 경우 str을 그대로 반환한다.
String padStart(String str, int minLength, char padChar)str의 길이가 minLength가 되도록 padChar을 앞 쪽에 채워넣은 문자열을 반환한다.
String padEnd(String str, int minLength, char padChar)str의 길이가 minLength가 되도록 padChar을 뒤 쪽에 채워넣은 문자열을 반환한다.
String repeat(String str, int count)str을 count만큼 반복하여 붙인 문자열을 반환한다.
