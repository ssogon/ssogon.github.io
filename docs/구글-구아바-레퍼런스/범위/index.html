범위

범위 개념을 직관적으로 다룰 수 있는 guava의 Range<C>에 대해 설명한다.

정의​

범위는 그 시작과 끝을 나타내는 양 끝점을 가지고 있으며, 이는 열릴 수도 닫힐 수도 있다.

열린 구간(open interval)은 끝점의 값을 포함하지 않으며, 소괄호로 표기한다.
닫힌 구간(closed interval)은 끝점의 값을 포함하며, 대괄호로 표기한다.

  • (a..b) = {x | a < x < b}
  • [a..b] = {x | a <= x <= b}
  • [a..b) = {x | a <= x < b}
  • (a..b] = {x | a < x <= b}
  • (a..+∞) = {x | x > a}
  • [a..+∞) = {x | x >= a}
  • (-∞..b) = {x | x < b}
  • (-∞..b] = {x | x <= b}
  • (-∞..+∞) = all values

끝점 b는 a보다 작아서는 안되며 최소 1개의 닫힌 구간이 있어야 동일할 수 있다.

  • [a..a]: 싱글톤 범위
  • [a..a); (a..a]: 비어있지만 유효함
  • (a..a): 유효하지 않음

Guava: Introduction

Range​

Range<C>는 C에 대한 범위를 가지는 불변 클래스다.

Range.closed("left", "right"); // all strings lexographically between "left" and "right" inclusive
Range.lessThan(4.0); // double values strictly less than 4

Guava Docs: Range

인스턴스 생성​

Range 인스턴스는 Range의 정적 메서드로 부터 얻을 수 있다.

정적 메서드설명
Range<C> open(C, C)(a..b)
Range<C> closed(C, C)[a..b]
Range<C> closedOpen(C, C)[a..b)
Range<C> openClosed(C, C)(a..b]
Range<C> singleton(C)[a..a]
Range<C> greaterThan(C)(a..+∞)
Range<C> atLeast(C)[a..+∞)
Range<C> lessThan(C)(-∞..b)
Range<C> atMost(C)(-∞..b]
Range<C> all()(-∞..+∞)

경계 유형을 명시적으로 전달하여 생성할 수도 있다.

정적 메서드설명
Range<C> range(C, BoundType, C, BoundType)a와 b의 경계 유형을 직접 지정한다.
Ranee<C> downTo(C, BoundType)a의 경계 유형을 직접 지정한다.
b는 양의 무한대를 가진다.
Ranee<C> upTo(C, BoundType)b의 경계 유형을 직접 지정한다.
a는 음의 무한대를 가진다.

Guava: Building Range

연산​

Range의 가장 기본적인 연산은 contains(C) 메서드다.

Range.closed(1, 3).contains(2); // returns true
Range.closed(1, 3).contains(4); // returns false
Range.lessThan(5).contains(5); // returns false
Range.closed(1, 4).containsAll(Ints.asList(1, 2, 3)); // returns true

Guava: Operations

조회 메서드​

Range는 현재 범위의 끝점을 조회하기 위한 메서드들을 가지고 있다.

  • hasLowerBound()와 hasUpperBound()는 범위에 지정된 끝점이 있는지, 아니면 무한대로 뻗어있는지 확인한다.
  • lowerBoundType()과 upperBoundType()은 끝점에 상응하는 BoundType을 반환한다.
    만약 지정된 끝점이 없다면 IllegalStateException이 발생한다.
  • lowerEndPoint()와 upperEndPoint()는 끝점의 값을 반환한다.
    만약 지정된 끝점이 없다면 IllegalStateException이 발생한다.
  • isEmpty()는 범위가 비어있는지 확인한다.
    이는 범위가 [a,a) 또는 (a,a]인지 확인하는 것과 같다.
Range.closedOpen(4, 4).isEmpty(); // returns true
Range.openClosed(4, 4).isEmpty(); // returns true
Range.closed(4, 4).isEmpty(); // returns false
Range.open(4, 4).isEmpty(); // Range.open throws IllegalArgumentException

Range.closed(3, 10).lowerEndpoint(); // returns 3
Range.open(3, 10).lowerEndpoint(); // returns 3
Range.closed(3, 10).lowerBoundType(); // returns CLOSED
Range.open(3, 10).lowerBoundType(); // returns OPEN

Guava: Query Operations

간격 메서드​

encloses​

가장 기본적인 관계 연산은 encloses(Range<C>)이며, 내부 범위가 외부 범위를 벗어나지 않는 경우 true를 반환한다.
이 연산은 각 끝점 간의 비교에만 전적으로 의존하기 때문에 사용시 주의가 필요하다.

  • [3..6]은 [4..5]를 둘러싼다.
  • (3..6)은 (3..6)을 둘러싼다.
  • [3..6]은 [4..4)를 둘러싼다. (후자가 비어있지만 끝점만 비교하기에 때문에)
  • (3..6]은 [3..6]을 둘러싸지 못한다.
  • [4..5]는 (3..6)을 둘러싸지 못한다. (동일한 범위이지만 끝점만 비교하기 때문에)
  • [3..6]는 (1..1]을 둘러싸지 못한다.

Guava: encloses

isConnected​

isConnected(Range<C>)는 두 범위가 연결되어 있는지 확인한다.

Range.closed(3, 5).isConnected(Range.open(5, 10)); // returns true
Range.closed(0, 9).isConnected(Range.closed(3, 4)); // returns true
Range.closed(0, 5).isConnected(Range.closed(3, 9)); // returns true
Range.open(3, 5).isConnected(Range.open(5, 10)); // returns false
Range.closed(1, 5).isConnected(Range.closed(6, 10)); // returns false

Guava: isConnected

intersection​

intersection(Range<C>)는 두 범위의 교집합을 반환한다.
교집합이 존재하지 않는다면 IllegalArgumentException을 발생시킨다.

Range.closed(3, 5).intersection(Range.open(5, 10)); // returns (5, 5]
Range.closed(0, 9).intersection(Range.closed(3, 4)); // returns [3, 4]
Range.closed(0, 5).intersection(Range.closed(3, 9)); // returns [3, 5]
Range.open(3, 5).intersection(Range.open(5, 10)); // throws IAE
Range.closed(1, 5).intersection(Range.closed(6, 10)); // throws IAE

Guava: intersection

span​

span(Range<C>)는 두 범위를 둘러싸는 최소 범위를 반환한다.
두 범위가 연결되어 있지 않아도 되지만, 연결되어 있다면 합집합으로 볼 수 있다.

Range.closed(3, 5).span(Range.open(5, 10)); // returns [3, 10)
Range.closed(0, 9).span(Range.closed(3, 4)); // returns [0, 9]
Range.closed(0, 5).span(Range.closed(3, 9)); // returns [0, 9]
Range.open(3, 5).span(Range.open(5, 10)); // returns (3, 10)
Range.closed(1, 5).span(Range.closed(6, 10)); // returns [1, 10]

Guava: span

DiscreteDomain​

Comparable 타입이 아닌 일부 타입들은 불연속적이며, 양쪽 경계가 지정된 범위로 열거할 수 있다.
DiscreteDomain<C>는 C에 대한 불연속적 도메인을 구현하며, 해당 타입의 전체 값 집합으로 표현된다.

인스턴스는 DiscreteDomain의 정적 메서드를 통해 생성할 수 있다.

정적 메서드설명
DiscreteDomain<Integer> integers()Integer에 대한 불연속 도메인을 생성한다.
DiscreteDomain<Long> longs()Long에 대한 불연속 도메인을 생성한다.
DiscreteDomain<BigInteger> bigIntegers()BigInteger에 대한 불연속 도메인을 생성한다.

생성된 DiscreteDomain<C>은 각종 범위 연산에 사용될 수 있다.

Guava: Discrete Domains

as ImmutableSortedSet​

ContiguousSet.create(Range<C>, DiscreteDomain<C>)는 DiscreteDomain<C>에서 Range<C>에 포함된 값들을 ImmutableSortedSet<C>로 반환한다.

ImmutableSortedSet<Integer> set = ContiguousSet.create(Range.open(1, 5), DiscreteDomain.integers());
// set contains [2, 3, 4]

ContiguousSet.create(Range.greaterThan(0), DiscreteDomain.integers());
// set domain [1, 2, ... Integer.MAX_VALUE]

as CanonicalForm​

range.canonical(DiscreteDomain<C>)는 DiscreteDomain<C>에서 Range<C>에 포함된 값들을 표준 형식의 Range<C>로 반환한다.
반환된 표준 형식 Range<C>는 아래 규칙들을 따른다.

  • 동등성: 모든 요소 v에 대해 a.canonical().contains(v) == a.contains(v)가 성립한다.
  • 유일성: a.isEmpty()가 아니라면 ContiguousSet.create(a, domain).equals(ContiguoutSet.create(b, domain))은 a.canonical(domain).equals(b.canonical(domain))을 의미한다.
  • 멱등성: a.canonical(domain).canonical(domain)은 a.canonical(domain)과 같다.

range.canonical(DiscreteDomain<C>)는 아래 표준 형식 중 하나를 반환함을 보장한다.

  • [start..end)
  • [start..+∞)
  • (-∞..end)
  • (-∞..+∞)

DiscreteDomain 커스텀​

DiscreteDomain<C>을 재정의하여 자신만의 DiscreteDomain<C>을 만들 수도 있지만, 몇 가지 명심해두어야할 사항이 있다.

  • DiscreteDomain<C>은 항상 해당 타입의 전체 값 집합으로 표현되어야 한다.
  • DiscreteDomain<C>은 무한할 수 있다.

Guava: Your Own DiscreteDomains

Comparator가 필요하다면​

Comparator가 필요하다면 다음 중 하나를 수행할 수 있다.

  • Range<C> 대신 Predicate<C>를 사용한다. Range<C>는 Predicate<C> 인터페이스를 구현하므로, Predicates.compose(Predicate<C>, Function<T, C>)의 첫 번째 인자에 Range<C>를 넘겨 Predicate<C>를 생성할 수 있다.
  • 원하는 순서를 정의한 객체를 감싼 래퍼 클래스를 사용한다.
