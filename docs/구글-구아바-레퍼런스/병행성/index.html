병행성

올바른 병행성 코드를 작성할 수 있도록 도와주는 guava의 병행성(concurrency) 유틸리티에 대해 설명한다.

ListenableFuture​

자바 8을 사용하는 경우 이 인터페이스는 더 이상 사용되지 않는다.
ListenableFuture<V>는 CompletableFuture<V>와 동일하게 동작한다.

Service​

Service는 상태를 가지는 객체를 표현하는 인터페이스다.
상태 로직 및 동기화 문제를 관리하기 위한 뼈대(skeleton)를 제공한다.

Guava: Service

생명 주기​

Service의 일반적인 생명 주기는 다음과 같으며, 한 번 중지된 서비스는 재시작될 수 없다.

  • Service.State.NEW
  • Service.State.STARTING
  • Service.State.RUNNING
  • Service.State.STOPPING
  • Service.State.TERMINATED

만약 서비스가 실행 도중 장애가 발생한다면 Service.State.FAILED 상태가 된다.

Guava: Using a Service

서비스 사용하기​

서비스는 Service.State.NEW 상태일 경우 startAsync() 메서드를 호출하여 비동기적으로 실행할 수 있다.
그리고 stopAsync() 메서드를 통해 해당 서비스를 종료할 수 있다.

stopAsync()는 startAsync()와는 달리 여러 번 호출되도 안전하다.
이는 서비스를 종료할 때 발생할 수 있는 경쟁 상태를 처리하기 위함이다.

서비스는 또한 서비스 전환이 완료되기를 기다리는 몇 가지 메서드들을 제공한다.

  • addListener() 메서드는 서비스의 상태가 변경될 때 호출되는 Service.Listener를 등록할 수 있다.
    실행 도중에 리스너를 추가하는 경우 이미 발생한 상태 전환에 대해서는 다시 실행되지 않는다.
  • awaitRunning() 메서드는 서비스가 Service.State.RUNNING 상태가 될 때까지 기다린다.
    이 메서드는 중간에 중단할 수 없으며 Checked Exception을 던지지도 않는다.
  • awaitTerminated() 메서드는 메서드가 Service.State.TERMINATED 또는 Service.State.FAILED 상태가 될 때까지 기다린다.
    이 메서드는 중간에 중단할 수 없으며 Checked Exception을 던지지도 않는다.

awaitRunning(), awaitTerminated() 메서드는 제한 시간을 지정할 수 있는 오버로딩도 있다.

구현체​

Service 인터페이스는 세밀하고 복잡하기 때문에 직접 구현하는 것은 추천하지 않는다.
대신 guava에서 제공하는 추상 클래스들을 기반으로 확장하여 사용할 것을 권장한다.

Guava: Implementations

AbstractIdleService​

AbstractIdleService는 Service.State.RUNNING 상태에서 아무런 작업도 하지 않는 서비스다.
아무 것도 하지 않기 때문에 스레드도 필요 없고, 단지 startUp(), shutDown() 메서드만 재정의하면 된다.

protected void startUp() {
  servlets.add(new GcStatsServlet());
}
protected void shutDown() {}

Guava: AbstractIdleService

AbstractExecutionThreadService​

AbstractExecutionThreadService는 단일 스레드에서 시작되고, 실행되고, 종료되는 서비스다.
실행 시 호출될 run() 메서드를 재정의해야 하며, 이는 중지 요청에 반응할 수 있어야 한다.

public void run() {
  while (isRunning()) {
    // perform a unit of work
  }
}

Guava: AbstractExecutionThreadService

AbstractScheduledService​

AbstractScheduledService는 주기적인 작업을 실행할 수 있는 서비스다.
작업의 한 반복을 지정하기 위해 runOneIteration() 메서드를 재정의하고, 실행 주기를 지정하기 위해 scheduler() 메서드를 재정의해야 한다.

protected void runOneIteration() {
  // running
}
protected Scheduler scheduler() {
  return Scheduler.newFixedRateSchedule(0, 1, TimeUnit.SECONDS);
}

Scheduler는 Scheduler.newFixedRateSchedule() 또는 Scheduler.newFixedDelaySchedule() 정적 메서드를 통해 생성할 수 있다.

  • rate는 작업의 완료 여부와는 관계 없는 주기적인 간격을 나타낸다.
  • delay는 한 작업이 완료된 후 다음 작업까지의 간격을 나타낸다.

Guava: AbstractScheduledService

AbstractService​

자신만의 수동 스레드를 관리해야 하는 경우 AbstractService를 직접 재정의할 수 있다.
재정의 해야 할 추상 메서드는 실행 및 종료 시 호출되는 doStart()와 doStop() 메서드다.

  • doStart() 메서드는 초기화를 수행한 다음 notifyStarted() 메서드를 호출해야 한다.
    만일 도중에 실패한 경우에는 notifyFailed() 메서드를 호출해야 한다.
  • doStop() 메서드는 종료 작업을 수행한 다음 notifyStopped() 메서드를 호출해야 한다.
    만일 도중에 실패한 경우에는 notifyFailed() 메서드를 호출해야 한다.

서비스 매니저​

ServiceManager는 여러 Service를 보다 쉽게 관리할 수 있는 방법을 제공한다.

Set<Service> services = ...;
ServiceManager manager = new ServiceManager(services);
manager.addListener(new Listener() {
  public void stopped() {}
  public void healthy() {
    // Services have been initialized and are healthy, start accepting requests...
  }
  public void failure(Service service) {
    // Something failed, at this point we could log it, notify a load balancer, or take
    // some other action. For now we will just exit.
    System.exit(1);
  }
}, MoreExecutors.directExecutor());

Runtime.getRuntime().addShutdownHook(new Thread() {
  public void run() {
    // Give the services 5 seconds to stop to ensure that we are responsive to shutdown
    // requests.
    try {
      manager.stopAsync().awaitStopped(5, TimeUnit.SECONDS);
    } catch (TimeoutException timeout) {
      // stopping timed out
    }
  }
});
manager.startAsync(); // start all the services asynchronously

인스턴스 메서드​

인스턴스 메서드설명
ServiceManager startAsync()관리중인 모든 서비스를 실행한다.
ServiceManager stopAsync()관리중인 모든 서비스를 종료한다.
void addListener(Listener, Executor)상태 전환시 호출될 리스너를 등록한다.
void awaitHealthy()관리중인 모든 서비스가 Service.State.RUNNING 상태가 될 때까지 기다린다.
void awaitStopped()관리중인 모든 서비스가 Service.State.TERMINATED 또는 Service.State.FAILED 상태가 될 때까지 기다린다.
boolean isHealthy()관리중인 모든 서비스가 Service.State.RUNNING 상태인지 확인한다.
ImmutableSetMultimap<State, Service> servicesByState()상태별로 색인화된 서비스의 스냅샷을 반환한다.
ImmutableMap<Service, Long> startupTimes()서비스별로 시작까지 걸린 milliseconds를 반환한다.
