리플렉션

자바 리플렉션 API를 보다 쉽고 안전하게 사용할 수 있도록 도와주는 guava의 리플렉션 유틸리티들에 대해 설명한다.

TypeToken​

제네릭의 type erasure로 인해 런타임에는 제네릭의 Class 객체를 참조할 수 없다.
때문에 ArrayList<String>과 ArrayList<Integer>는 같은 ArrayList<Object>로 평가된다.

ArrayList<String> stringList = Lists.newArrayList();
ArrayList<Integer> intList = Lists.newArrayList();
System.out.println(stringList.getClass().isAssignableFrom(intList.getClass()));
// returns true, even though ArrayList<String> is not assignable from ArrayList<Integer>

TypeToken은 런타임에 제네릭 타입을 수정하거나 조회할 수 있는 리플렉션 기반 트릭을 제공한다.

Guava: TypeToken

배경지식​

제네릭은 JDK 1.5에서 도입되었고 이전 버전과의 호환성을 맞추기 위해 제네릭과 관련된 코드는 컴파일 타임에 지워져야 했다.
컴파일시 제네릭은 타입에 대한 안정성만을 검증받은 후 사라지기 때문에 런타임에는 제네릭 타입에 대한 정보를 알 수 없다.

그러나 리플렉션을 이용한다면 클래스나 메서드의 제네릭 타입을 감지할 수 있다.
예를 들어 리플렉션을 통해 List<String>을 반환하는 메서드의 반환 타입을 얻으면 List<String>을 나타내는 ParameterizedType이 반환된다.

TypeToken은 이러한 대안을 사용하여 제네릭 타입을 최소한의 오버헤드로 조작한다.

Guava: Background: Type Erasure and Reflection

소개​

기본 원시 클래스에 대한 TypeToken을 얻는 것은 간단하다.

TypeToken<String> stringTok = TypeToken.of(String.class);
TypeToken<Integer> intTok = TypeToken.of(Integer.class);

컴파일 타임에 제네릭 타입 인수를 알고 있으며, 제네릭 타입에 대한 TypeToken을 얻으려는 경우에는 빈 익명 내부 클래스를 사용하면 된다.

TypeToken<List<String>> stringListTok = new TypeToken<List<String>>() {};

만약 의도적으로 와일드카드 타입을 참조하려는 경우에는 다음과 같이 작성한다.

TypeToken<Map<?, ?>> wildMapTok = new TypeToken<Map<?, ?>>() {};

TypeToken은 동적으로 제네릭 타입 인수를 해결하는 방법을 제공하기도 한다.

static <K, V> TypeToken<Map<K, V>> mapToken(TypeToken<K> keyToken, TypeToken<V> valueToken) {
  return new TypeToken<Map<K, V>>() {}
      .where(new TypeParameter<K>() {}, keyToken)
      .where(new TypeParameter<V>() {}, valueToken);
}
...
TypeToken<Map<String, BigInteger>> mapToken = mapToken(
    TypeToken.of(String.class),
    TypeToken.of(BigInteger.class));
TypeToken<Map<Integer, Queue<String>>> complexToken = mapToken(
    TypeToken.of(Integer.class),
    new TypeToken<Queue<String>>() {});

where(TypeParameter<X>, TypeToken<X>)을 사용하지 않으면 실제로 K와 V에 할당된 타입을 구체화할 수 없다.

static <K, V> TypeToken<Map<K, V>> incorrectMapToken() {
  return new TypeToken<Map<K, V>>() {};
}
...
System.out.println(<String, BigInteger>incorrectMapToken());
// just prints out "java.util.Map<K, V>"

아래와 같이 자신의 제네릭 타입을 담는 TypeToken<X>를 필드로 두는 것도 좋은 방법이다.

abstract class IKnowMyType<T> {
  TypeToken<T> type = new TypeToken<T>(getClass()) {};
}
...
new IKnowMyType<String>() {}.type; // returns a correct TypeToken<String>

Guava: Introduction

조회 메서드​

TypeToken<X>은 Class를 지원하는 다양한 메서드들을 가지고 있다.

인스턴스 메서드설명
Type getType()java.lang.reflect.Type을 반환한다.
Class<T> getRawType()가장 잘 알려진 런타임 클래스를 반환한다.
TypeToken<T> getSubtype(Class<?>)this의 특정 하위 타입을 반환한다.
만약 this가 Iterable<String>이고 인자가 List.class인 경우 List<String>이 반환된다.
TypeToken<T> getSupertype(Class<?>)this의 특정 상위 타입을 반환한다.
만약 this가 Set<String>이고 인자가 Iterable.class인 경우 Iterable<String>이 반환된다.
boolean isSupertypeOf(TypeToken<X>)this가 특정 타입의 상위 타입인지 확인한다.
TypeSet getTypes()this와 this의 모든 하위 클래스 및 인터페이스가 담긴 TypeSet을 반환한다.
boolean isArray()int[]나 <? extends A[]>와 같은 배열 타입인지 확인한다.
TypeToken<?> getComponentType()int[]나 <? extends A[]>와 같은 배열 컴포넌트의 타입을 반환한다.
배열 컴포넌트가 아니라면 null이 반환된다.

Guava: Queries

resolveType​

resolveType() 메서드는 컨텍스트 토큰의 타입 인자를 대체할 수 있는 강력하지만 복잡한 쿼리 메서드다.

TypeToken<Function<Integer, String>> funToken = new TypeToken<Function<Integer, String>>() {};

TypeToken<?> funResultToken = funToken.resolveType(Function.class.getTypeParameters()[1]));
// returns a TypeToken<String>

일반적으로 특정 메서드의 반환 타입을 추론하는 데 사용된다.

TypeToken<Map<String, Integer>> mapToken = new TypeToken<Map<String, Integer>>() {};
TypeToken<?> entrySetToken = mapToken.resolveType(Map.class.getMethod("entrySet").getGenericReturnType());
  // returns a TypeToken<Set<Map.Entry<String, Integer>>>

Guava: resolveType

Invokable​

Invokable은 java.lang.reflect.Method와 java.lang.reflect.Constructor를 보다 사용하기 쉽게 래핑한 클래스다.

Guava: Invokable

정적 메서드​

정적 메서드설명
Invokable<T,​ T> from​(Constructor<T>)특정 생성자의 Invokable을 반환한다.
Invokable<?, ​Object> from​(Method)특정 메서드의 Invokable을 반환한다.

인스턴스 메서드​

인스턴스 메서드설명
boolean isPublic()접근 지정자가 public인지 확인한다.
boolean isProtected()접근 지정자가 protected인지 확인한다.
boolean isPrivate()접근 지정자가 private인지 확인한다.
boolean isPackagePrivate()접근 지정자가 디폴트인지 확인한다.
boolean isStatic()static으로 선언되었는지 확인한다.
boolean isFinal()final으로 선언되었는지 확인한다.
boolean isOverridable()오버라이딩이 가능한지 확인한다.
boolean isAnnotationPresent(Class<Annotation>)특정 어노테이션이 있는지 확인한다.
TypeToken<R> getReturnType()반환 타입을 조회한다.
ImmutableList<Parameter> getParameters()선언된 모든 매개변수를 조회한다.

다이나믹 프록시​

Reflection​

Reflection는 단일 인터페이스 타입의 다이나믹 프록시를 생성하는 안전하고 편리한 API다.

Foo foo = Reflection.newProxy(Foo.class, invocationHandler);

Guava: newProxy

AbstractInvocationHandler​

때로는 다이나믹 프록시가 보다 직관적인 방법으로 equals(), hashCode(), toString()를 지원하기를 원할 수도 있다.
프록시 인스턴스는 인터페이스 타입이 동일하고 호출 핸들러가 같은 경우 다른 프록시 인스턴스와 동일하다 볼 수 있다.
따라서 프록시의 toString()은 보다 쉽게 커스터마이징할 수 있도록 호출 핸들러의 toString()에 위임할 수 있다.

AbstractInvocationHandler는 이러한 아이디어를 구현한다.

Guava: AbstractInvocationHandler

ClassPath​

ClassPath는 가장 효율적인 클래스 경로 탐색을 제공하는 유틸리티로, jar 파일 또는 파일 시스템 디렉토리 아래의 클래스만을 스캔한다.

ClassPath classpath = ClassPath.from(classloader); // scans the class path used by classloader
for (ClassPath.ClassInfo classInfo : classpath.getTopLevelClasses("com.mycomp.mypackage")) {
  ...
}

Guava: ClassPath

클래스 로딩​

Reflection.initialize(Class<?>...)는 지정한 클래스들을 초기화한다. (모든 정적 초기화가 수행된다.)

정적 상태는 시스템 유지보수와 테스트 가능성을 손상시키기 때문에 이 메서드를 사용하는 것은 코드 스멜을 만드는 것과 같다.
레거시 프레임워크와 상호 운용해야 하는 상황이며 선택의 여지가 없는 경우에만 고려해보자.
