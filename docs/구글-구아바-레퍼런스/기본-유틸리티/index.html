기본 유틸리티

범용적으로 사용될 수 있는 guava의 기본 유틸리티에 대해 설명한다.

Optional​

Optional<T>는 null일 수도 있는 T에 대한 참조를 나타낸다.

Optional<Integer> possible = Optional.of(5);
possible.isPresent(); // return true
possible.get(); // return 5

com.google.common.base.Optional은 java.util.Optional과 매우 비슷하지만 양측 모두 상위 타입을 제공하지 않아 서로 호환되지는 않는다.
guava 팀은 이 클래스를 @Deprecated할 계획은 없지만 가능한한 자바 표준을 사용할 것을 권장하고 있다.

클래스설명
java.util.Optional구현한 메서드가 더 많으며, OptionalInt와 같은 primitive type을 위한 Optional도 제공한다.
com.google.common.base.OptionalSerializable을 구현해 직렬화가 가능하다.

Guava: Optional
Guava Docs: Optional

팩토리 메서드​

팩토리 메서드설명
Optional<T> of(T)T를 참조하는 Optional을 생성한다.
T가 null이라면 NullPointerException이 발생한다.
Optional<T> fromNullable(T)T를 참조하는 Optional을 생성한다.
T가 null이라도 예외가 발생하지 않는다.
Optional<T> absent()참조를 가지지 않는 빈 Optional을 생성한다.

Guava: Making an Optional

인스턴스 메서드​

인스턴스 메서드설명
boolean isPresent()Optional이 참조하는 인스턴스가 null이 아니라면 true를 반환한다.
T get()Optional이 참조하는 인스턴스를 반환한다.
참조하는 인스턴스가 null이라면 IllegalStateException이 발생한다.
T or(T)Optional이 참조하는 인스턴스를 반환한다.
참조하는 인스턴스가 null이라면 매개변수로 지정한 기본 값을 반환한다.
T orNull()Optional이 참조하는 인스턴스를 반환한다.
참조하는 인스턴스가 null이라면 그대로 null을 반환한다.

Guava: Optional query methods

Preconditions​

Preconditions는 여러가지 전제 조건을 확인하는 유틸리티 클래스다.
guava 팀은 이를 정적으로 import하여 사용할 것을 권장한다.

import static com.google.common.base.Preconditions.checkArgument;

Guava: Preconditions

정적 메서드​

정적 메서드설명실패 시 발생되는 예외
void checkArgument(boolean)매개변수 boolean이 true인지 확인한다.IllegalArgumentException
T checkNotNull(T)매개변수 T가 null이 아닌지 확인한다.
T를 그대로 반환하기 때문에 인라인하게 작성할 수 있다.		NullPointerException
void checkState(boolean)매개변수 boolean이 true인지 확인한다.IllegalStateException
void checkElementIndex(int index, int size)매개변수 index가 0 이상 size 미만인지 확인한다.IndexOutOfBoundsException
void checkPositionIndex(int index, int size)매개변수 index가 0 이상 size 이하인지 확인한다.IndexOutOfBoundsException
void checkPositionIndexes(int start, int end, int size)매개변수 start와 end가 0 이상 size 이하인지 확인한다.IndexOutOfBoundsException

각 메서드는 세 가지의 변형(override)을 가지고 있다.

  1. 추가 매개변수가 없는 경우, 모든 예외는 오류 메시지 없이 발생한다.
  2. 추가 매개변수가 Object인 경우, 오류 메세지는 object.toString()이 된다.
  3. 추가 매개변수가 임의 개수의 String인 경우, printf(String, Object...)처럼 동작하는데 GWT 호환성과 효율성을 위해 "%s" 구분자만을 사용한다.
    참고로, checkNotNull, checkArgument, checkState는 auto boxing을 피하기 위해서 가변 매개변수가 아닌 primary type을 사용하는 여러가지 오버로드를 제공한다.

다음은 세 번째 변형의 예시다.

checkArgument(i >= 0, "Argument was %s but expected nonnegative", i);
checkArgument(i < j, "Expected i < j, but %s >= %s", i, j);

Guava: Preconditions

Verify​

Preconditions가 생성자 및 메서드에 유효하지 않은 인자가 전달됐는지 확인하는 용도라면 Verify는 의도하지 않은 상황으로 흘러가는 것을 막기 위해 사용된다.
항상 활성화되어 있다는 것만을 제외하면 자바의 assert문과 유사하다고 볼 수 있겠다.

Verify.verify(bill.status() == Status.UNPAID,
    "Unexpected bill status: %s", bill.status());

Guava Docs: Verify

정적 메서드설명실패 시 발생되는 예외

T verifyNotNull(T)매개변수 T가 null이 아닌지 확인한다.
T를 그대로 반환하기 때문에 인라인하게 작성할 수 있다.		VerifyException

각 메서드는 Preconditions와 같은 변형(override)을 가지고 있다.

Ordering의 대부분의 기능들은 이제 Stream과 Comparator에서 제공된다.

Guava: Ordering
Guava Docs: Ordering

ComparsionChain​

ComparsionChain은 여러 개의 compare를 게으르게(lazy) 수행하는 비교자다.
이 비교는 0이 아닌 결과를 찾을 때까지만 수행되며, 그 뒤의 비교는 평가되지 않는다.

public int compareTo(Foo that) {
  return ComparisonChain.start()
      .compare(this.aString, that.aString)
      .compare(this.anInt, that.anInt)
      .compare(this.anEnum, that.anEnum, Ordering.natural().nullsLast())
      .result();
}

Guava: compare/compareTo

Objects​

자바 7 이상을 사용하는 경우 이 클래스는 더 이상 사용되지 않는다.
java.util.Objects는 com.google.common.base.Objects와 동등한 메서드를 제공한다.

Guava 21.0에서 Objects의 몇몇 메서드는 아예 제거되기도 했다.

Guava: Object common methods

MoreObjects​

MoreObjects는 모든 Object에 적용할 수 있는 헬퍼 메서드를 제공한다.

정적 메서드설명
T firstNonNull(T first, T second)두 매개변수 중 null이 아닌 첫 번째 매개변수를 반환한다.
만약 둘다 null이라면 NullPointerException을 발생시킨다.
String toStringHelper(Class<?>)toString()의 반환 값을 쉽게 생성할 수 있도록 도와준다.

다음은 toStringHelper(Class<?>)를 사용하는 예이다.

// Returns "ClassName{x=1}"
MoreObjects.toStringHelper(this)
    .add("x", 1)
    .toString();

Guava: Convenience methods

Throwables​

Throwables는 예외를 단순화하는 방법을 제공한다.

Guava: Throwables

전파​

때때로 잡은 Exception을 더 상위의 try-catch 블록으로 위임해야할 때가 있다.
Throwables는 예외 전파를 단순하는 몇 가지 유틸리티 메서드를 제공한다.

정적 메서드설명
void propagateIfPossible(Throwable, Class<X extends Throwable>)매개변수 Throwable이 RuntimeException, Error, X 중 하나인 경우 예외를 그대로 던진다.
void throwIfInstanceOf(Throwable, Class<X extends Exception>)매개변수 Throwable이 X인 경우 예외를 그대로 던진다.
void throwIfUnchecked(Throwable)매개변수 Throwable이 RuntimeException, Error 중 하나인 경우 예외를 그대로 던진다.

다음은 예외를 전파하는 간단한 예시다.

try {
  someMethodThatCouldThrowAnything();
} catch (IKnowWhatToDoWithThisException e) {
  handle(e);
} catch (Throwable t) {
  Throwables.throwIfInstanceOf(t, IOException.class);
  Throwables.throwIfInstanceOf(t, SQLException.class);
  Throwables.throwIfUnchecked(t);
  throw new RuntimeException(t);
}

Guava: Propagation

원인 체인​

Throwables는 예외의 원인(cause) 체인을 참조하는 여러 메서드들을 제공한다.

정적 메서드설명
Throwable getRootCause(Throwable)발단이 되는 원인을 반환한다.
List<Throwable> getCausalChain(Throwable)원인 체인을 List 형태로 반환한다.
String getStackTraceAsString(Throwable)stacktrace를 String으로 반환한다.

Guava: Casual Chain

Stopwatch​

Stopwatch는 경과된 시간을 간단하게 측정할 수 있는 기능을 제공한다.

Stopwatch stopwatch = Stopwatch.createStarted();
doSomething();
stopwatch.stop(); // optional
Duration duration = stopwatch.elapsed();
log.info("time: " + stopwatch); // formatted string like "12.3 ms"

thread-safe하게 설계되진 않았으니 스레드간 공유해서는 안된다.

Guava Docs: Stopwatch

팩토리 메서드​

팩토리 메서드설명
Stopwatch createStarted()System.nanoTime()를 사용하는 StopWatch를 생성하고 시작시킨다.
Stopwatch createUnstarted()System.nanoTime()를 사용하는 StopWatch를 생성하고 시작시키지는 않는다.

Guava Docs: Stopwatch

인스턴스 메서드​

인스턴스 메서드설명
Duration elapsed()경과된 시간을 Duration으로 반환한다.
long elapsed(TimeUnit)경과된 시간을 원하는 TimeUnit 단위로 반환한다.
boolean isRunning()스톱워치가 시간을 재고 있는지 확인한다.
Stopwatch reset()경과된 시간을 0으로 만들고 멈춘다.
Stopwatch start()스톱워치를 시작시킨다.
Stopwatch stop()스톱워치를 멈춘다.
String toString()경과된 시간을 "12.3 ms"와 같은 포맷으로 반환한다.

Stopwatch::toString이 반환하는 문자열은 최대 4자리의 소수점을 가지며, 이에 따라 단위 문자열이 "d", "h", "min", "s", "ms", "μs", "ns" 중 하나로 바뀔 수 있다.

Guava Docs: Stopwatch
