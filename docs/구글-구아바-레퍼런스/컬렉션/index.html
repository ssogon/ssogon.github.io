컬렉션

JDK 컬렉션 생태계를 확장한 guava의 컬렉션에 대해 설명한다.

불변 컬렉션​

객체의 불변 복사본을 만드는 것은 방어적 프로그래밍의 기법 중 하나다.
guava는 표준 컬렉션 및 구아바 컬렉션의 불변 버전을 제공하고 있다.

public static final ImmutableSet<Color> GOOGLE_COLORS =
    ImmutableSet.<Color>builder()
        .addAll(WEBSAFE_COLORS)
        .add(new Color(0, 191, 255))
        .build();

다음은 Immutable*이 제공하는 static 메서드들이다.

메서드설명
Immutable*<E> of(E...)매개변수 E...를 원소로 가지는 불변 객체를 생성한다.
Immtuable*.Builder builder()불변 객체를 만드는 빌더를 반환한다.
Immutable*<E> copyOf(Collection<E>)매개변수 Collection<E>의 불변 복사본을 반환한다.

만약 JDK 9 이상을 사용한다면 Collection의 팩토리 메서드를 사용하자.

List<String> names = List.of("scott", "michael", "holden");

Guava: Immutable Collections

구아바 컬렉션​

guava에는 JDK에 없는 여러 가지 컬렉션들이 포함되어 있다.
이들은 JDK 컬렉션 추상화에 묶이지 않으면서도 서로 상생할 수 있도록 설계되었다.

구아바 컬렉션은 공통적으로 create() 정적 메서드로 인스턴스를 생성한다는 특징이 있다.

Guava: New Collection Types

Multiset​

Multiset<E>은 요소별 개수를 세는 인터페이스다.

Multiset<String> multiset = HashMultiset.create();
multiset.add("a");
multiset.add("a");
multiset.add("b");
multiset.count("a"); // 2

SortedMultiset​

SortedMultiset<E>는 특정 범위의 sub-multiset을 효율적으로 가져올 수 있는 subMultiset(...) 메서드를 가지고 있다.

SortedMultiset<String> treeMultiset = TreeMultiset.create();
treeMultiset.add("a");
treeMultiset.add("a");
treeMultiset.add("b");
treeMultiset.subMultiset("a", BoundType.CLOSED, "c", BoundType.CLOSED).size(); // 3

Guava: SortedMultiset

Multimap​

MultiMap<K, V>은 하나의 key에 여러 value가 매핑될 수 있는 컬렉션이다.
Map<K, List<V>>를 사용하기 쉽게 만든 것이라 생각하면 된다.

Multimap<String, Integer> multimap = ArrayListMultimap.create();
multimap.put("k", 1);
multimap.put("k", 2);
multimap.put("k", 3);
multimap.get("k"); // [1, 2, 3]

빌더​

Multimap<K, V>을 생성하는 가장 쉬운 방법은 MultimapBuilder를 사용하는 것이다.
아래와 같이 key-value 구조를 어떻게 표현할 것인지를 지정할 수 있다.

// creates a ListMultimap with tree keys and array list values
ListMultimap<String, Integer> treeListMultimap =
    MultimapBuilder.treeKeys().arrayListValues().build();

// creates a SetMultimap with hash keys and enum set values
SetMultimap<Integer, MyEnum> hashEnumMultimap =
    MultimapBuilder.hashKeys().enumSetValues(MyEnum.class).build();

구현 클래스의 create() 정적 메서드로 직접 생성할 수도 있다.

Multimap<String, String> multimap = ArrayListMultimap.create();

MultiMap은 Map이 아니다​

Multimap<K, V>은 Map<K, V>을 구현하지 않았을 뿐더러 동작 방식도 조금 상이하다.

  • Multimap.get(K)는 null 대신 빈 Collection<V>을 반환한다.
  • Multimap.size()는 key의 개수가 아닌 entry의 개수를 반환한다.

만약 Map<K, V>과 유사한 동작을 원한다면 asMap() 메서드를 사용해야 한다.

Guava: Multimap Is Not A Map

BiMap​

BiMap<K, V>은 key와 value의 고유성을 동시에 보존하는 양방향 Map<K, V>이다.
양방향성을 보장하기 때문에 key와 value를 서로 반전시키는 것이 가능하다.

BiMap<String, Integer> nameToId = HashBiMap.create();
nameToId.put("Bob", 42);
BiMap<Integer, String> idToName = nameToId.inverse(); // {42=Bob}

Table​

Table<R, C, V>은 row와 column을 사용하는 테이블 구조를 나타내기 위한 클래스다.

Table<Vertex, Vertex, Double> weightedGraph = HashBasedTable.create();
weightedGraph.put(v1, v2, 4);
weightedGraph.put(v1, v3, 20);
weightedGraph.put(v2, v3, 5);

weightedGraph.row(v1); // returns a Map mapping v2 to 4, v3 to 20
weightedGraph.column(v3); // returns a Map mapping v1 to 20, v2 to 5

ClassToInstanceMap​

ClassToInstanceMap<B>은 타입에 따라 구현체를 매핑하는 Map<Class<B>, B>다.
캐스팅 과정을 신경쓰지 않으면서 type-safety하게 제어하는 방법을 제공한다.

ClassToInstanceMap<Number> numberDefaults = MutableClassToInstanceMap.create();
numberDefaults.putInstance(Integer.class, 1);
numberDefaults.putInstance(Double.class, 2.0);

numberDefaults.getInstance(Integer.class); // 1

RangeSet​

RangeSet<C>은 복수의 Range<C>를 합치거나 제외하며 범위를 표현할 수 있는 클래스다.
각 범위는 서로 연결될 수 있으면 병합되고, 빈 범위는 무시된다.

RangeSet<Integer> rangeSet = TreeRangeSet.create();
rangeSet.add(Range.closed(1, 10)); // {[1, 10]}
rangeSet.add(Range.closedOpen(11, 15)); // disconnected range; {[1..10], [11..15)}
rangeSet.add(Range.closedOpen(15, 20)); // connected range; {[1..10], [11..20)}
rangeSet.add(Range.openClosed(0, 0)); // empty range; {[1..10], [11..20)}
rangeSet.remove(Range.open(5, 10)); // splits [1..10]; {[1..5], [10..10], [11..20)}

rangeSet.complement(); // [(-∞..1), (5..10), (10..11), [20..+∞)]
rangeSet.span(); // [1..20)

Guava: RangeSet

RangeMap​

RangeMap<K, V>는 RangeSet<C>과는 달리 인접한 범위라도 병합되지 않는다.

RangeMap<Integer, String> rangeMap = TreeRangeMap.create();

rangeMap.put(Range.open(3, 6), "bar"); // {[1..3]=foo, (3..6)=bar}
rangeMap.put(Range.open(10, 20), "foo"); // {[1..3]=foo, (3..6)=bar, (10..20)=foo}
rangeMap.remove(Range.closed(5, 11)); // {[1..3]=foo, (3..5)=bar, (11..20)=foo}

팩토리 메서드​

각 유틸리티 클래스의 new*(T...) 메서드를 사용하면 Collection을 선언과 동시에 초기화할 수 있다.

Set<Type> copySet = Sets.newHashSet(elements);
List<String> theseElements = Lists.newArrayList("alpha", "beta", "gamma");

Collection의 크기를 지정하여 생성하는 방법도 제공한다.

List<Type> exactly100 = Lists.newArrayListWithCapacity(100);
List<Type> approx100 = Lists.newArrayListWithExpectedSize(100);
Set<Type> approx100Set = Sets.newHashSetWithExpectedSize(100);

참고로 guava에서 제공하는 Collection들은 생성자를 노출하지 않거나 유틸리티 클래스에 초기화 메서드가 없다.
대신 다음과 같이 해당 Collection에 정적 팩토리 메서드를 직접 노출하고 있다.

Multiset<String> multiset = HashMultiset.create();

Iterables​

guava는 가능한 Collection보다는 더 범용적인 Iterable을 위한 유틸리티를 만드는 것을 선호한다.
그렇기에 Collection에 대해 지원될 것이라 기대되는 많은 메서드들은 대부분 Iterables에서 찾을 수 있을 것이다.

Iterable<Integer> concatenated = Iterables.concat(
    Ints.asList(1, 2, 3),
    Ints.asList(4, 5, 6));
// concatenated has elements 1, 2, 3, 4, 5, 6

String lastAdded = Iterables.getLast(myLinkedHashSet);

String theElement = Iterables.getOnlyElement(thisSetIsDefinitelyASingleton);
// if this set isn't a singleton, something is wrong!

Multimap<K, V>은 여러 키를 하나의 값에 매핑하거나 하나의 키를 여러 값에 매핑할 수 있으므로 이를 반전시키는 것이 유용할 수 있다.

// note that we choose the implementation, so if we use a TreeMultimap, we get results in order
/*
 * inverse maps:
 *  1 => {"a"}
 *  2 => {"a", "b", "c"}
 *  3 => {"c"}
 *  4 => {"a", "b"}
 *  5 => {"c"}
 *  6 => {"b"}

Forwarding Decorators​

모든 Collection에 대해 데코레이터 패턴을 쉽게 사용할 수 있도록 Forwarding* 추상 클래스가 제공된다.
Forwarding* 추상 클래스를 상속하여 서브클래싱하고 데코레이팅된 객체를 반환하는 delegate() 추상 메서드를 재정의하기만 하면 모든 메서드를 직접 위임할 필요 없이 데코레이팅된 기능을 추가할 수 있다.

class AddLoggingList<E> extends ForwardingList<E> {
  final List<E> delegate; // backing list
  @Override protected List<E> delegate() {
    return delegate;
  }
  @Override public void add(int index, E elem) {
    log(index, elem);
    super.add(index, elem);
  }
  @Override public boolean add(E elem) {
    return standardAdd(elem); // implements in terms of add(int, E)
  }
  @Override public boolean addAll(Collection<? extends E> c) {
    return standardAddAll(c); // implements in terms of add
  }
}

PeekingIterator​

PeekingIterator는 next()를 호출하지 않고도 다음 요소를 엿볼 수 있는 Iterator의 하위 클래스다.
Iterators의 peekingIterator(Iterator) 메서드로 Iterator를 PeekingIterator로 래핑할 수 있다.

List<E> result = Lists.newArrayList();
PeekingIterator<E> iter = Iterators.peekingIterator(source.iterator());
while (iter.hasNext()) {
  E current = iter.next();
  while (iter.hasNext() && iter.peek().equals(current)) {
    // skip this duplicate element
    iter.next();
  }
  result.add(current);
}

Guava: PeekingIterator

AbstractIterator​

AbstractIterator 추상 클래스를 이용하면 자신만의 Iterator를 쉽게 구현할 수 있다.
구현해야할 추상 메서드는 computeNext() 단 하나로, 다음 값을 반환해준다.
만약 시퀀스가 완료됐다면 endOfData()를 반환하면 된다.

public static Iterator<String> skipNulls(final Iterator<String> in) {
  return new AbstractIterator<String>() {
    protected String computeNext() {
      while (in.hasNext()) {
        String s = in.next();
        if (s != null) {
          return s;
        }
      }
      return endOfData();
    }
  };
}

Guava: AbstractIterator

AbstractSequentialIterator​

AbstractSequentialIterator는 자신만의 Iterator를 더 쉽게 구현할 수 있는 방법을 제공한다.
해당 computeNext()는 초기 값을 추가로 전달해야 하며, 종료되어야 하는 경우 null을 반환한다.

Iterator<Integer> powersOfTwo = new AbstractSequentialIterator<Integer>(1) { // note the initial value!
  protected Integer computeNext(Integer previous) {
    return (previous == 1 << 30) ? null : previous * 2;
  }
};
