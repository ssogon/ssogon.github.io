해싱

여러가지 해싱 방법을 제공하는 guava의 Hashing에 대해 설명한다.

Guava: Hashing Overview

Hashing​

Hashing은 HashFunction을 생성하는 팩토리 메서드를 가지고 있다.

HashFunction hashFunction = Hashing.md5();
HashCode hashCode = hashFunction.newHasher()
    .putLong(id)
    .putString(name, Charsets.UTF_8)
    .putObject(person, personFunnel)
    .hash();

Guava: Hashing

정적 메서드​

정적 메서드설명
HashFunction md5()MD-5 알고리즘을 구현하는 HashFunction을 반환한다.
HashFunction crc32()CRC-32 알고리즘을 구현하는 HashFunction을 반환한다.
HashFunction sha256()SHA-256 알고리즘을 구현하는 HashFunction을 반환한다.
HashFunction hmacSha256(byte[])SHA-256 알고리즘과 특정 비밀 키를 사용하는 HashFunction을 반환한다.
HashFunction hmacSha256(Key)SHA-256 알고리즘과 특정 비밀 키를 사용하는 HashFunction을 반환한다.

Guava: Provided Hash Functions

HashFunction​

HashFunction은 특정 해시 알고리즘을 나타내는 stateless 클래스다.

Guava: HashFunction

인스턴스 메서드​

인스턴스 메서드설명
Hasher newHasher()새로운 Hasher 인스턴스를 반환한다.
HashCode hashBytes(byte[] input)newHasher().putBytes(input).hash()의 shortcut이다.
HashCode hashString(CharSequence input, Charset charset)newHasher().putString(input, charset).hash()의 shortcut이다.
HashCode hashUnencodedChars(CharSequence input)newHasher().putUnencodedChars(input).hash()의 shortcut이다.
HashCode hashObject(T instance, Funnel<T> funnel)newHasher().putObject(instance, funnel).hash()의 shortcut이다.

Guava Docs: HashFunction

Hasher​

Hasher는 임의의 데이터 블록을 해싱할 수 있는 stateful 클래스다.

Guava: Hasher

인스턴스 메서드​

인스턴스 메서드설명
HashCode hash()현재 해시에 들어있는 값을 기반으로 HashCode를 계산하여 반환한다.
putBoolean(boolean)현재 해시에 특정 boolean 값을 넣는다.
putBytes(byte[])현재 해시에 특정 byte[] 값을 넣는다.
putChar(char)현재 해시에 특정 char 값을 넣는다.
putDouble(double)현재 해시에 특정 double 값을 넣는다.
putFloat(float)현재 해시에 특정 float 값을 넣는다.
putInt(int)현재 해시에 특정 int 값을 넣는다.
putLong(long)현재 해시에 특정 long 값을 넣는다.
putObject(T, Funnel<T>)현재 해시에 특정 T 값을 넣는다.
T를 파싱할 때 사용할 Funnel<T>가 필요하다.
putShort(short)현재 해시에 특정 short 값을 넣는다.
putString(CharSequence, Charset)현재 해시에 특정 String 값을 넣는다.
String을 디코딩할 때 사용할 Charset이 필요하다.
putUnencodedChars(CharSequence)CharSequence의 각 char 값을 순서대로 처리하는 것과 같다.

Guava Docs: Hasher

HashCode​

HashCode는 해싱 결과를 나타내는 도메인이다.

Guava: HashCode

정적 메서드​

정적 메서드설명
HashCode fromBytes(byte[])byte[]로 부터 HashCode를 생성한다.
HashCode fromInt(int)int로 부터 32비트 HashCode를 생성한다.
HashCode fromLong(long)long으로 부터 64비트 HashCode를 생성한다.
HashCode fromString(String)16진수로 인코딩된 String으로 부터 HashCode를 생성한다.

Guava Docs: HashCode

인스턴스 메서드​

인스턴스 메서드설명
byte[] asBytes()해시 값을 byte[] 형태로 반환한다.
int asInt()해시 값의 처음 4바이트를 리틀 엔디안 순서의 int 값으로 변환하여 반환한다.
long asLong()해시 값의 처음 8바이트를 리틀 엔디안 순서의 long 값으로 변환하여 반환한다.
int bits()해시 값의 비트 수를 반환한다. (8의 배수)
String toString()해시 값의 각 바이트를 2자리의 16진수 String으로 변환하여 반환한다.

Guava Docs: HashCode

Funnel​

Funnel은 특정 객체를 primitive 값들로 분해하는 방법을 정의하는 클래스이다.
void funnel(T, PrimitiveSink) 메서드를 구현해야 한다.

Funnel<Person> personFunnel = new Funnel<Person>() {
  @Override
  public void funnel(Person person, PrimitiveSink into) {
    into.putInt(person.id)
        .putString(person.firstName, Charsets.UTF_8)
        .putString(person.lastName, Charsets.UTF_8)
        .putInt(birthYear);
  }
};

참고로, putString("abc", Charsets.UTF_8).putString("def", Charsets.UTF_8)과 putString("ab", Charsets.UTF_8).putString("cdef", Charsets.UTF_8)은 동일한 바이트 배열을 만들어내기 때문에 의도치 않은 해시 충돌이 발생할 수 있다.
이는 각 필드 사이에 구분자 값을 추가함으로써 방지할 수 있는 문제다.

Guava: Funnel

BloomFilter​

BloomFilter는 확률성 자료 구조인 블룸 필터를 구현한 클래스다.

BloomFilter<Person> friends = BloomFilter.create(personFunnel, 500, 0.01);
for (Person friend : friendsList) {
  friends.put(friend);
}
// much later
if (friends.mightContain(dude)) {
  // the probability that dude reached this place if he isn't a friend is 1%
  // we might, for example, start asynchronously loading things for dude while we do a more expensive exact check
}
